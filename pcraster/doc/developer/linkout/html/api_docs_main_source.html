113  \section xmlDocumentation The XML vocabulary
114 
115  A single XML vocabulary defined in PCRaster.xsd is used in two roles (in and out):
116  - in: pcr_createScriptFromXMLFile(), pcr_createScriptFromXMLString() to create a script.
117  - out: pcr_ScriptXmlReflection() to retrieve meta information from a script.
118 
119  Within the PCRaster.xsd schema the XML nodes are documented within an
120  xs:annotation element (view XML source in Visual Studio).
121  The difference in application of the [in] and [out] case of certain nodes is
122  clearly stated. Note that not all nodes present in PCRaster.xsd are used in the current API.
123 
124  \section errorHandling Error Handling
125 
126  All functions will store a possible error description in the PcrScript object.
127  The success of all functions can be checked of the function call by evaluating
128  the result of pcr_ScriptError() and/or pcr_ScriptErrorMessage().
129  Errors on a PcrScript object are NOT recoverable; All successive calls on
130  the same PcrScript will do nothing. A PcrScript object in error state should
131  still be deleted with pcr_destroyScript().
